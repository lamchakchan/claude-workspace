# Project Instructions

## Project
Name: claude-workspace (Claude Code Platform Engineering Kit)
Purpose: Preconfigured AI agent platform for teams adopting Claude Code
Tech Stack: Go, Shell scripts, YAML, JSON, Markdown
Build: `go build ./...`
Test: `go test ./...`

## Team Conventions

### Code Style
- Go: Follow standard Go conventions (gofmt, effective Go)
- Shell scripts: Use `set -euo pipefail`, quote variables, use shellcheck
- JSON: 2-space indentation, no trailing commas
- Markdown: ATX headings, fenced code blocks with language tags

### Testing
- All hooks must be tested before deployment
- Scripts should handle edge cases gracefully
- Validate with `shellcheck` for shell scripts

### Documentation
- Every new feature needs a corresponding docs update
- Configuration changes must be reflected in README.md
- Use inline comments for non-obvious logic

## Directory Layout

```
_template/.claude/    - Embeddable template assets (agents, skills, hooks, settings)
_template/.mcp.json   - Embeddable MCP config template
.claude/              - Active project config (generated by attach, customized)
.mcp.json             - Active MCP config (generated by attach)
main.go               - Go CLI entry point
assets.go             - Embed directive for _template/
internal/attach/      - attach command implementation
internal/doctor/      - doctor command implementation
internal/mcp/         - mcp command implementation
internal/platform/    - Shared platform utilities (FS, ExtractTo, etc.)
internal/sandbox/     - sandbox command implementation
internal/setup/       - setup command implementation
internal/upgrade/     - upgrade command implementation
scripts/              - Build and release scripts
```

## Important Files

- `README.md` - Main documentation and quick start guide
- `_template/.claude/settings.json` - Template settings with safe defaults
- `_template/.mcp.json` - Template MCP server configurations
- `.claude/settings.json` - Active project settings
- `.mcp.json` - Active MCP config
